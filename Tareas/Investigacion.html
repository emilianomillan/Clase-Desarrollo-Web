Etiquetas HTML y Elementos de Página

Introducción

En el desarrollo web actual, una correcta comprensión de las etiquetas HTML va más allá de su apariencia visual. Es fundamental entender su significado y cómo este afecta a la accesibilidad (la capacidad de que todos los usuarios puedan navegar el sitio) y al posicionamiento en buscadores web. La diferencia entre una etiqueta que solo modifica la apariencia y una que aporta un significado específico al contenido es clave. A la par, para herramientas como los lectores de pantalla, o para los algoritmos de Google, esta distinción es esencial para interpretar correctamente la estructura y el propósito de una página.


Diferencias de Significado y Apariencia en Etiquetas HTML

Dentro de HTML, existen etiquetas diseñadas para organizar el contenido. Es cotidiano encontrar pares de etiquetas que, aunque producen un resultado visual idéntico, poseen en realidad significados y funciones distintas. Comprender esta diferencia es importante para los motores de búsqueda y para la correcta interpretación de una página, ya que un contenido bien estructurado con los significados adecuados es más fácil de indexar y clasificar.

Etiquetas <b> vs. <strong>

Aunque ambas etiquetas muestran el texto en negrita, su propósito es diferente.

La etiqueta <b> (de bold) tiene una función solamente de estilo. Su propósito es destacar visualmente una porción del texto para llamar la atención del lector, sin añadirle una mayor importancia. Su uso es comparable a aplicar el formato de negrita en un procesador de textos solo para que algo resalte.

En cambio, la etiqueta <strong> indica al navegador y a los buscadores que el texto contenido es de gran importancia, seriedad o urgencia. Un lector de pantalla podría, por ejemplo, cambiar su entonación al leer este texto para reflejar dicha importancia. Para Google, el uso de <strong> en palabras clave puede ser una señal de que ese contenido es especialmente relevante para el tema de la página.

Etiquetas <i> vs. <em>

De forma similar, estas dos etiquetas suelen mostrar el texto en cursiva, pero su intención es distinta.

La etiqueta <i> (de italic) se utiliza para diferenciar un texto del resto por razones de estilo, como al señalar un término técnico, el título de una obra, una frase en otro idioma o un pensamiento. No implica que el texto tenga un énfasis especial. Por ejemplo: "El término Homo sapiens se refiere a la especie humana".

La etiqueta <em> (de emphasis o énfasis) se utiliza para aplicar un énfasis que modifica la interpretación de una frase. Como ejemplo, la frase: "Yo nunca me robaría esa dona". El énfasis en "nunca" altera por completo el mensaje, y <em> es la etiqueta adecuada para comunicar esa intención.

Etiquetas <s> vs. <del>

Ambas etiquetas tachan el texto, pero comunican ideas diferentes.

La etiqueta <s> (de strikethrough) se usa para marcar información que ya no es correcta o relevante, pero que se mantiene visible por razones de contexto. Un caso de uso común se encuentra en las compras por internet: "Precio: $50.00 $25.00". El precio original ya no es válido, pero se muestra para que el usuario perciba el descuento.

La etiqueta <del> (de delete) se utiliza para indicar que un texto fue eliminado de un documento. Su contraparte es la etiqueta <ins> (de insert), que marca contenido que ha sido añadido. El uso conjunto de estas etiquetas es muy útil para mostrar revisiones en un texto, dejando un registro claro de lo que se ha quitado y lo que se ha agregado.


Elementos de Línea y Elementos de Bloque

En HTML, todas las etiquetas se clasifican en dos categorías principales según su comportamiento en la página: elementos "de línea" y elementos "de bloque". Esta diferencia define cómo se organizan visualmente.

Elementos de Línea

Los elementos de línea (inline) ocupan únicamente el espacio horizontal que su contenido requiere y no inician una nueva línea. Se integran en el flujo normal del texto, permitiendo que otros elementos se coloquen a un lado.

Ejemplos de Elementos de Línea: <a>, <strong>, <em>, <img> (imágenes), <span>.

Elementos de Bloque

Los elementos de bloque (block) funcionan de manera opuesta. Siempre comienzan en una nueva línea y ocupan todo el ancho disponible de su "contenedor". Esto crea una estructura de "cajas" apiladas verticalmente, que son las piezas fundamentales para construir el diseño de una página.

Ejemplos de Elementos de Bloque: <div>, <p>, <h1>-<h6> (todos los títulos), <ul>, <li>.


Formularios y Etiquetas en HTML

Los formularios son la principal herramienta para que los usuarios interactúen y envíen información en una página web. La etiqueta <form> actúa como el contenedor que agrupa todos los controles (campos, botones, etc.) y gestiona el envío de los datos a un servidor. Este proceso se configura con sus atributos action (la URL a la que se envían los datos) y method (el método de envío, como GET o POST).

Etiqueta <input> y su Propiedad type

Su función cambia por completo según el valor de su atributo type.

Ejemplos son:

  • type="text": Un campo de texto de una sola línea para datos generales.
  • type="password": Similar al de texto, pero los caracteres se ocultan por seguridad.
  • type="submit": Un botón que, al ser presionado, envía los datos del formulario.
  • type="checkbox": Una casilla de verificación que permite al usuario seleccionar una o varias opciones.
  • type="radio": Un botón de opción. En un grupo con el mismo nombre, solo se puede seleccionar uno.
  • type="date": Muestra una interfaz de calendario para seleccionar una fecha.
  • type="file": Permite al usuario seleccionar un archivo de su dispositivo para cargarlo.

Etiqueta <label>

La etiqueta es fundamental para la "usabilidad" y accesibilidad de los formularios. Asocia un texto descriptivo a un campo del formulario a través del atributo for. Esto ofrece dos ventajas:

  • Accesibilidad: Un lector de pantalla leerá el texto de la etiqueta cuando el usuario enfoque el campo, indicando qué información se debe introducir.
  • Usabilidad: Permite a los usuarios hacer clic en el texto de la etiqueta para activar el campo asociado, lo que facilita la interacción, especialmente en dispositivos móviles.

Etiqueta <output>

Esta etiqueta se utiliza para mostrar el resultado de un cálculo o una acción del usuario dentro del formulario. Su propósito es indicar que su contenido es un resultado generado por la página.

Etiqueta <progress>

Muestra una barra que representa visualmente el progreso de una tarea. Utiliza dos atributos principales: max, que define el valor total (por ejemplo, 100), y value, que indica el estado actual del progreso.

Etiquetas <select> y <option>

La etiqueta crea un menú desplegable. Dentro de ella, cada etiqueta <option> define una de las opciones de la lista. El atributo value de cada <option> contiene el dato que se enviará al servidor, mientras que el texto visible para el lector es el que se encuentra dentro de la etiqueta.

Etiqueta <textarea>

A diferencia de <input type="text">, esta etiqueta crea un campo de texto de múltiples líneas. Es ideal para que los usuarios introduzcan comentarios, mensajes o cualquier texto de mayor longitud. Su tamaño inicial se puede definir con los atributos rows (filas) y cols (columnas).
